                  W7D3: Journaling through Backbone

                  Today we created a small journal web app that allows you to make and delete notes within a quick backbone app. As usual, pictures and trial app to come!

                  Clarifications from class:

                  • The Backbone default class is sufficiently featured so there should not be any need to overwrite the base class.
                  • In the Backbone view (and only the Backbone view!), there are certain attributes that will be set automatically: model, collection, el, id, className, tagName, attributes and events. If these attributes are passed in when creating the Backbone view, they do not need to be set as attributes in an initialize method. 
                    
// The postView instance will have the postView.model attribute already defined.
var postView = new JournalApp.Views.PostsIndexItem({model: post})

// The initialize method is unnecessary
JournalApp.Views.PostsIndex = Backbone.View.extend({
  initialize: function(options){
    this.model = options.model
  },
});
                  • Use event handlers (in an event hash so that they are automatically removed when a view is deleted) to begin refreshes or create changes based on user interaction. #listenTo events to know when to refresh the view automatically based on server or model/collection changes.
                  • If there is no “{trigger: true}” in the Backbone.history.navigate, then the route is not actually triggered.
                  • When defining templates on the Backbone view, you only need to define JST[‘template/url’]. The “JST[“templateName”] = _.template($(“#template-tag”).html())” is automatically defined when creating a typical Backbone app.

                  Tips from Class

                  • The extensions for making backbone templates is template_name.jst.ejs (name + “just” + “EJ’s”). Backbone will also change the url slightly (backbone views are have a # before the url: http://www.myapp.com/#posts/11).
                  • Minimize the logic within the views. This will make troubleshooting easier, since it it remarkably difficult to troubleshoot html code.
                  • Prefer #content versus #html, #escape versus #get. This will prevent javascript inject attacks.
                  • If you get a strange error about nokogiri-1.6.6.2 not being able to install, try the following steps:

                  1) $ bundle config build.nokogiri —use-system-libraries.

                  2) $ bundle install

                  Style Advice (in no particular order)

                  • HTML uses hyphen-case to name variables, javascript uses camelCase, and Ruby uses snake_case. These simple naming conventions can get confusing when switching between different languages.
                  • In Backbone, private methods are prefixed with an underscore. Like in Ruby, these methods are not actually protected in any way, they’re just an indication to readers of the code that the method should not be used.

                  Points to Note from Reading for Tomorrow: Today’s solutions

                  • A useful method for a collection is a #getOrFetch method, which will either get the model with the requested id, or it will fetch the information from the database.
                  • The #create method in the Rails Controller should render a json object if it succeeds, and render a hash of the errors and unprocessable_entity status if it fails. 
                    
  def create
    @model = Model.new(post_params)
    if @model.save
      render :json => @model
    else
      render :json => @model.errors.full_messages, status: :unprocessable_entity
    end
  end
                  • To use the serializeJSON gem, download the file “jquery.serializeJSON.js”, store it in vendor > assets > javascript and then require it in app > assets > javascripts > application.js. This gem is used to serialize the input name and values from a form. They can then be passed directly into the save function.
                  • When writing a success and error callback, consider defining the functions as variables and then using these variables as callbacks within the function. It makes the code easier to read.
                  • To make the form prettier, focus on inputs when clicking into a form field.
                  • Pass in the element to be edited within the app > assets > javascript > app_name.js file under the initialize method. Then, initialize the function:
                    
$(document).ready(function(){
  JournalApp.initialize();
});

                    W7D2: Creating Routers and Views for Pokemon Browser

                    Today we continued to exploring backbone— specifically how backbone routers can be used to quickly direct users to specific views. This adds another layer on top of the standard rails app, which allows you to create quick navigations that do not render new pages every time a user makes a selection.

                    I’ll post my demo site once I’m on my own computer, but below are some screenshots of what our page looks like. Every third of a page can be refreshed without navigating to new urls, but we also made it so navigating to a specific url will activate the backbone router, which will generate a backbone view that is used to populate the page.

                    Clarifications from class:

                    • The app > assets > javascript > application.js file uses a gem (sprockets) that uses manifest files to include javascript files within the current folder, the vendor > javascript folder, and the installed gems. The manifest file recognizes the filenames from the comments. The same type of assets pipeline is used for stylesheets. This assets pipeline is a unique feature/benefit of rails.
                    • Using a “require tree” in the application.js file can be convenient, but files will be loaded in unordered manner. If each file is “require”ed individually, the load order will be the order of the files listed.
                    • The parse function is used in Backbone models because many times, backbone developers do not have control over the format they get information from the api. In the case of the pokemon app, parse is used to convert a json array into an array of Backbone models.
                    • Backbone collections accept both models and options, the first parameter passed in always accepts a model. To just specify options, pass in an empty array.
                    • Code blocks in the json builder can be useful if multiple attributes need to be set.
                    • If a route uses the singular “resource”, define a url for that Backbone model. If a route uses the plural “resources”, define a urlRoot for that Backbone model.
                    • Backbone views are great because they use an events hash to define the event handlers (makes defining multiple event handlers very easy). 
                      
Pokedex.Views.PokemonIndex = Backbone.View.extend({
  events: {

    var $target = $(event.currentTarget);

                    Tips from Class

                    • Use json builders to specify conditions when data will be received by Backbone. This way, information is not processed or retrieved from the database unnecessarily (that will slow down your app!)
                    • New views will accept an options hash with a model: and collection:. If present, the model: and collection: for a view will be automatically set to the #model and #collection attribute (no need for an initialize method).
                    • When passing in a callback, #bind(this) to the function to avoid defining a “that” variable. Otherwise, if callbacks use methods defined on the current object, the method needs to be called on a “that” variable, which is a pointer to the object (“this”).
                    • In backbone, this.$(selector) is shorthand for this.$el.find(selector).
                    • Don’t replace this.$el in a Backbone view. This will remove the event handlers from the view’s element.
                    • If Backbone.history.navigate (which is not documented, but takes a url path and an options hash) is called, {trigger: true} needs to be added as part of the options hash or else any newly added element will not have event handlers enabled and the redirect will not actually redirect to the correct page.

                    • Don’t append anything to jQuery objects selected from the page while in a Backbone view method. Do this in the Backbone router.
                    • Don’t use in-line css styling. The styling will be hard-coded into the webpage and won’t be changeable through the css files. For long input buttons, put the first attribute on the same line as the element tag.
